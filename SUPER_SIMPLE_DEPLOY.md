## 📦 ШАГ 1: Загрузка файлов на GitHub (5 минут)

GitHub — это облачное хранилище для кода. Как Google Drive, но для программ.

### 1.1 Зайдите на сайт
Откройте браузер и перейдите по адресу:
```
https://github.com/signup
```

### 1.2 Создайте аккаунт
Нажмите зелёную кнопку **"Create an account"**

Заполните:
- **Email** — ваша почта
- **Password** — придумайте пароль (сложный!)
- **Username** — ваш ник (например: `vasya-smart`)

[КАРТИНКА: Страница регистрации GitHub]
```
┌─────────────────────────────────────┐
│  Join GitHub                        │
│                                     │
│  Email: [your@email.com        ]    │
│  Password: [••••••••••••       ]    │
│  Username: [vasya-smart        ]    │
│                                     │
│  [🟢 Create account]               │
└─────────────────────────────────────┘
```

### 1.3 Подтвердите почту
GitHub отправит письмо — откройте его и нажмите подтверждающую ссылку.

---

### 1.4 Создайте новый репозиторий (папку для проекта)

После входа вы увидите страницу. Найдите зелёную кнопку:

[КАРТИНКА: Кнопка создания репозитория]
```
┌─────────────────────────────────────┐
│  ➕ New repository                   │
│     (зелёная кнопка слева)         │
└─────────────────────────────────────┘
```

Нажмите на **"Create repository"** или **"New"**

Заполните форму:

[КАРТИНКА: Форма создания репозитория]
```
┌─────────────────────────────────────┐
│  Create a new repository            │
│                                     │
│  Repository name:                   │
│  [smart-goals-app            ]     │
│                                     │
│  ☑️ Public (видно всем)            │
│  ☐ Private (только вам)            │
│                                     │
│  ☐ Add a README file               │
│  ☐ Add .gitignore                  │
│  ☐ Choose a license                │
│                                     │
│  [🟢 Create repository]            │
└─────────────────────────────────────┘
```

Введите:
- **Repository name:** `smart-goals-app`
- **Public:** поставьте галочку ✅
- Остальное НЕ трогайте
- Нажмите **Create repository**

---

### 1.5 Загрузите файлы проекта

После создания репозитория вы увидите страницу.

Прокрутите немного вниз и найдите ссылку:

[КАРТИНКА: Ссылка для загрузки]
```
┌─────────────────────────────────────┐
│  ...or upload an existing file      │
│  🔗 uploading an existing file     │
└─────────────────────────────────────┘
```

Нажмите на **"uploading an existing file"**

Теперь у вас есть два варианта:

---

#### Вариант А: Загрузка ZIP-архива (ПРОЩЕ)

**Шаг А1:** Создайте ZIP-архив с проектом

На вашем компьютере:
1. Откройте папку `smart-goals-app`
2. Выделите ВСЕ файлы (Ctrl+A)
3. Правой кнопкой → **"Отправить"** → **"Сжатая ZIP-папка"**
4. Получится файл `smart-goals-app.zip`

**Шаг А2:** Загрузите на GitHub

На странице загрузки GitHub:
1. Нажмите **"choose your files"**
2. Выберите созданный ZIP-файл
3. Подождите загрузки
4. Прокрутите вниз
5. В поле **"Commit changes"** напишите: `initial commit`
6. Нажмите **"Commit changes"**

---

#### Вариант Б: Загрузка папкой (если ZIP не работает)

**Шаг Б1:** Перетащите файлы

На странице загрузки GitHub:
1. Откройте папку `smart-goals-app` на компьютере
2. Выделите ВСЕ файлы внутри (Ctrl+A)
3. Перетащите их мышкой в браузер, на страницу GitHub
4. Отпустите мышку — файлы начнут загружаться
5. Подождите (может занять 1-2 минуты)
6. Прокрутите вниз
7. В поле **"Commit changes"** напишите: `initial commit`
8. Нажмите **"Commit changes"**

---

## 🔧 ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

### Проблема: "Build Failed" (Ошибка сборки)

**Решение:**
1. На странице Vercel нажмите на красную надпись **"Failed"**
2. Прокрутите вниз до красного текста
3. Скопируйте текст ошибки
4. Отправьте мне — я исправлю!

### Проблема: "404 Not Found" (Страница не найдена)

**Решение:**
1. Подождите 2 минуты (иногда нужно время)
2. Обновите страницу (F5)
3. Если не помогло — напишите мне

### Проблема: "Unauthorized" (Нет доступа)

**Решение:**
1. Перейдите в настройки проекта (Settings)
2. Проверьте Environment Variables
3. Убедитесь, что все 3 переменные заполнены правильно
4. Пересоберите проект (см. ниже)

## 🔄 КАК ПЕРЕСОБРАТЬ ПРОЕКТ

Если вы что-то исправили и хотите обновить сайт:

4. Нажмите на неё
5. Нажмите кнопку **"Redeploy"**

Или просто измените любой файл на GitHub — Vercel сам пересоберёт!
